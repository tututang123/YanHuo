# 工作 Work

这里记录工作模式、会议、决策、沟通、项目经验和职业成长。

## 使用方式

工作上的事情可以直接交给 AI 处理。默认做法是：

1. 原始信息先放进 `../00-inbox`，不要急着整理。
2. AI 根据内容分流到任务、会议、沟通、决策、项目或复盘。
3. 需要继续推进的事情写入 `task-board.md`。
4. 已经形成结论的事情写入 `decision-log.md`、`meeting-notes.md` 或具体项目。
5. 完成、暂停或不再活跃的内容归档到 `../90-archive`。

## 工作笔记

- `communication.md`：沟通
- `meeting-notes.md`：会议记录
- `career-growth.md`：职业成长
- `decision-log.md`：决策记录
- `work-patterns.md`：工作方法
- `task-board.md`：任务看板
- `ai-workflow.md`：AI 代办与归档规则

## 安全边界

- 密码、密钥、客户敏感信息、公司未授权材料不要直接写进仓库。
- 必须记录时，只写摘要、代号、结论和下一步，把原始敏感材料放到本地私有位置。
- 需要 AI 处理敏感内容时，先脱敏再交给 AI。
